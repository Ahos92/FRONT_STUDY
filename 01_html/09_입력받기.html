# Input 태그를 이용해 사용자로부터 다양한 입력을 받을 수 있다.

  • 사용자의 입력들은 폼(form)단위로 묶어놓아야 한다.
  • 해당 form 내부에서 submit이 발생하면 데이터를 전송한다.
  • input태그를 form태그의 외부에서 추가할 수도 있다.
  • ★★★★★ 데이터 전송시 input태그의 name 속성이 key 역할을 한다.
  • ★★★★★ 데이터 전송시 input태그에 사용자가 입력한 내용이 value 역할을 한다.
  • input태그의 value속성이 value역할을 할 때도 있다.
Type 모양 설명
type="text" 일반적인 텍스트를 입력하는 칸
type="password" 입력하는 내용이 가려지는 칸
type="submit" 사용자가 input태그에 입력해놓은 데이터를 전송하는 버튼
type="reset" 사용자가 input태그에 입력해놓은 데이터를 초기화하는 버튼
type="radio" name속성이 같을 때 여러가지 중 하나만 선택할 수 있는 버튼
type="checkbox" 동시에 여러가지를 선택할 수 있는 버튼
type="email" 해당 텍스트가 이메일인지 검사해준다.
<select> country: 사용자가 여러 옵션 중 하나를 선택할 수 있다.
<textarea> 많은 양의 텍스트를 입력할 때 사용

※ GET방식에는 전송 데이터 길이제한이 있으므로
POST방식을 사용하는 것이 좋다.
type="color" 색상 선택 (ex : #ffffff)
type="date" 날짜 선택 (ex : 2020-12-31)
type="search" 검색어를 입력하는 필드를 생성한다.
type="tel" 전화번호를 입력하는 필드를 생성한다.
type="url" url을 입력하는 필드를 생성한다.
※ 특정 기기는 url속성이 들어간 입력란에
알아서 .com을 붙여주기도 한다.
type="range" volume :
  • checked : 페이지가 로드되었을때 미리 체크해놓는다.
  • disabled : 해당 필드를 사용할 수 없게 된다 (전송에서 제외)
  • readonly : 해당 필드를 수정할 수 없게 된다 (전송에는 포함)
  • required : 해당 필드의 값을 반드시 채워야 한다.
  • pattern : 해당 필드에 정규표현식 검사를 할 수 있다.
  • size : 입력칸의 크기(너비)를 설정할 수 있다.
  • value : 값을 지정해놓을 수 있다.
  • autofocus : 자동으로 포커싱되게 할 수 있다.
  • autocomplete : on/off 입력칸 자동완성 가능 여부
  • maxlength : 최대 길이를 설정
  • min : 최소값 설정
  • max : 최대값 설정
  • step : 값의 단위를 설정
# min/max/step Example
quantity (value 20 / min 20 / max 100 / step 10)
